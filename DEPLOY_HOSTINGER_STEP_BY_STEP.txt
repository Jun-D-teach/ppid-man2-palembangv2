PANDUAN DEPLOY PPID MAN 2 PALEMBANG KE HOSTINGER
=================================================

PAKET INI SUDAH DISIAPKAN UNTUK HOSTINGER.
Jangan upload folder node_modules. Hostinger akan menjalankan npm install sendiri.

A. SETTING NODE.JS APP DI HOSTINGER
-----------------------------------
1. Masuk hPanel Hostinger.
2. Pastikan paket hosting mendukung Node.js Application.
3. Buka menu Website / Advanced / Node.js atau Web Apps.
4. Buat aplikasi Node.js baru.
5. Upload file ZIP project ini lalu extract.
6. Application Root: pilih folder project ini, folder yang berisi package.json utama.
   Jangan pilih folder backend saja.
7. Startup file: backend/server.js
8. Build command: npm run build
9. Start command: npm start
10. Node.js version: pilih versi LTS yang tersedia, disarankan Node 20 atau Node 22.

B. DATABASE MYSQL
-----------------
1. Di hPanel, buka Databases / MySQL Databases.
2. Buat database baru.
3. Catat data berikut:
   - DB_HOST
   - DB_USER
   - DB_PASSWORD
   - DB_NAME
4. Buka phpMyAdmin.
5. Pilih database yang baru dibuat.
6. Import file db_ppid_man2.sql.
7. Jika tabel menu_pages dan menu_page_files belum ada, import juga MIGRASI_MENU_CMS.sql.
   Biasanya tidak wajib karena server.js juga membuat tabel CMS otomatis saat server jalan.

C. ENVIRONMENT VARIABLES
------------------------
Isi Environment Variables di Node.js App Hostinger:

NODE_ENV=production
DB_HOST=isi_host_database_hostinger
DB_USER=isi_user_database_hostinger
DB_PASSWORD=isi_password_database_hostinger
DB_NAME=isi_nama_database_hostinger
JWT_SECRET=ganti_dengan_kalimat_rahasia_panjang
JWT_EXPIRES_IN=1d
MAX_UPLOAD_SIZE=10485760
CORS_ORIGIN=https://domainkamu.com

Catatan:
- Jika memakai subdomain, contoh: https://ppid.man2plg.sch.id
- Jangan isi VITE_API_URL untuk production satu domain. Biarkan kosong/tidak dibuat.
- Jangan gunakan DB_HOST localhost kecuali Hostinger memang menampilkan localhost.

D. SETELAH DEPLOY
-----------------
1. Buka domain.
2. Tes halaman utama.
3. Tes login admin.
4. Tes menu Atur Menu Website.
5. Tes upload file.
6. Tes klik file upload dari halaman publik.
7. Segera ganti password admin default.

E. KALAU ERROR
--------------
1. Kalau halaman putih: cek build command npm run build.
2. Kalau API 500: cek environment database.
3. Kalau 401 Unauthorized: logout, hapus localStorage, login ulang.
4. Kalau upload tidak bisa dibuka: pastikan folder backend/uploads ada dan bisa ditulis.
5. Kalau database kosong: ulang import db_ppid_man2.sql di phpMyAdmin.
